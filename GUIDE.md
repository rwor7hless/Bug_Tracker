# Руководство для модераторов

## Телеграм-бот

Напиши `/start` боту, чтобы открыть главное меню. Всё управление — через кнопки в чате.

### Создание репорта

1. Нажми **📝 Создать репорт**
2. Выбери категорию:
   - **Краш** — вылет / крэш клиента или сервера
   - **Лаги** — просадки FPS, тик-рейта, задержки
   - **Визуал** — графические артефакты, неправильные текстуры
   - **Геймплей** — сломанная механика, неработающий функционал
   - **Другое** — всё остальное
   - **📝 Предложение** — идея по улучшению
3. Введи **название** (до 100 символов) — кратко опиши суть
4. Введи **описание** — подробности, шаги для воспроизведения
5. Бот проверит, нет ли **похожих тикетов**. Если есть — можешь добавить bump к существующему вместо создания дубликата
6. Укажи **срочность**: 🟢 Обычная / 🟡 Высокая / 🔴 Критичная
7. Приложи **лог/краш-репорт** (ссылка, текст или файл) — или пропусти
8. Приложи **скриншоты** (до 10 фото) — или пропусти

Тикету автоматически присваивается тег (например `CRH-001`, `LAG-012`).

### Что такое Bump?

**Bump** = «Я тоже это встретил». Это счётчик, показывающий сколько людей столкнулось с одной и той же проблемой. Чем больше бампов — тем выше приоритет тикета для разработчиков.

Как добавить bump:
1. В меню нажми **📋 Тикеты** → **Bump**
2. Введи тег тикета (например `BUG-001`)
3. Bump засчитан

Также bump предлагается автоматически, если при создании нового репорта бот найдёт похожие тикеты.

### Просмотр тикетов

Нажми **📋 Тикеты** — откроется список открытых тикетов, сгруппированных по категориям. Доступна навигация по страницам и поиск по тегу или тексту.

### Вход в веб-панель

1. Нажми **🔑 Пароль для панели**
2. Бот выдаст логин (твой @username в Telegram) и одноразовый пароль
3. Зайди на сайт и введи эти данные
4. При следующем входе запроси пароль заново

## Веб-панель

На сайте можно:
- **Просматривать тикеты** с фильтрацией по статусу, категории, срочности
- **Искать** по тегу или тексту (семантический поиск)
- **Создавать тикеты** — с автоматической проверкой на дубликаты
- **Добавлять bump** к существующим тикетам
- **Переключаться** между вкладками «Баги» и «Предложения»

Разработчики оставляют комментарии к тикетам — уведомление придёт тебе в Telegram.

### Статусы тикетов

| Статус | Значение |
|--------|----------|
| **Открыт** | Новый, ещё не рассмотрен |
| **В работе** | Разработчик занимается |
| **Дубликат** | Совпадает с другим тикетом |
| **Решён** | Исправлен |
